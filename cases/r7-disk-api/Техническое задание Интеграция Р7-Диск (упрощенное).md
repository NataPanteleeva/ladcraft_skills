# Техническое задание (упрощённая версия)

**Разработка навыка интеграции с Р7-Диск для платформы Ladcraft**

| | |
|---|---|
| **Версия документа** | 1.0 |
| **Дата** | 02.06.2026 |
| **Платформа** | Ladcraft (AI-агент) |
| **Внешняя система** | Р7-Диск, REST API KS 2024 |

---

## 1. Цель разработки

Разработать навык для AI-агента, который обеспечивает безопасную и удобную работу с файлами и папками Р7-Диска из чата, без ручных REST-запросов.

Навык должен:

- подключаться к серверу Р7-Диск по учетным данным;
- показывать содержимое папок и структуру каталогов;
- выполнять базовые операции с папками и файлами;
- создавать документы с текстом напрямую из чата;
- корректно обрабатывать кириллицу в именах и содержимом;
- явно сообщать пользователю об ограничениях API.

---

## 2. Область применения

Решение предназначено для пользователей и администраторов Ladcraft, которым требуется:

- работать с файловым хранилищем Р7-Диска через диалог с агентом;
- использовать единый набор операций на разных стендах;
- быстро переключаться между сохранённым подключением и другим сервером;
- получать стандартизированные ответы по результатам операций.

Навык не включает разработку внешних прикладных подсистем и локальных интерфейсов вне платформы Ladcraft.

---

## 3. Функциональные требования

### 3.1. Подключение и авторизация

Система должна поддерживать:

- авторизацию по API Р7-Диска;
- хранение параметров подключения в настройках навыка;
- кэширование токена сессии для повторного использования;
- обязательное уточнение у пользователя: использовать сохранённые данные или подключиться к другому серверу.

### 3.2. Работа с папками

Система должна обеспечивать:

- просмотр содержимого выбранной папки;
- рекурсивный просмотр структуры каталогов;
- создание, перемещение, копирование, удаление и восстановление папок.

Ограничение: переименование папок через API недоступно, система должна возвращать понятное сообщение о невозможности этого действия.

### 3.3. Работа с файлами

Система должна обеспечивать:

- создание файлов в папке;
- загрузку и скачивание файлов;
- перемещение, копирование, удаление и восстановление файлов;
- переименование файлов;
- проверку существования файла по имени;
- получение версий файла и переключение версии;
- конвертацию документа в поддерживаемые форматы.

### 3.4. Работа с текстом и DOCX

Система должна поддерживать создание документов из текстового запроса пользователя:

- простой текст UTF-8;
- DOCX-документы с базовым форматированием (жирный/курсив);
- переносы строк как отдельные абзацы.

Сложное форматирование (таблицы, стили, шаблоны) выполняется через загрузку заранее подготовленного файла.

---

## 4. Требования к данным и настройке

Для работы навыка должны задаваться:

- базовый адрес API сервера Р7-Диск;
- логин;
- пароль;
- (опционально) идентификатор корневой папки по умолчанию.

Должно быть разделение между веб-адресом интерфейса и API-адресом сервера.

---

## 5. Требования к безопасности

Должны быть обеспечены:

- использование HTTPS;
- хранение чувствительных данных в защищённых настройках;
- запрет вывода секретов в ответы пользователю;
- выполнение операций в рамках прав авторизованного пользователя;
- корректная обработка ошибок доступа и истечения токена.

---

## 6. Нефункциональные требования

- Предсказуемые и человекочитаемые ответы агента.
- Корректная работа с русским языком и Unicode.
- Совместимость со стендами, использующими KS 2024 API.
- Возможность расширения набора операций при обновлении API.

---

## 7. Ограничения и допущения

- Поддерживаются только возможности, доступные в API KS 2024.
- Переименование папок не поддерживается API и не реализуется как операция.
- Для нестандартных сценариев допускается работа через веб-интерфейс Р7-Диска.

---

## 8. Ожидаемый результат

По итогам разработки должен быть поставлен рабочий навык, который позволяет пользователю из чата:

- подключаться к Р7-Диску;
- просматривать и управлять папками/файлами;
- создавать документы по текстовому запросу;
- получать понятные ответы о результате операции и причинах ограничений.

---

## 9. Критерии приёмки

1. Успешная авторизация и получение рабочей сессии.
2. Корректный просмотр папок и файлов на тестовом стенде.
3. Успешное выполнение операций с файлами и папками в рамках API.
4. Корректная обработка кириллицы в именах и содержимом.
5. Создание DOCX из текста с базовым форматированием.
6. Понятное сообщение о невозможности переименования папки.
7. Отсутствие утечки секретных данных в ответах.
