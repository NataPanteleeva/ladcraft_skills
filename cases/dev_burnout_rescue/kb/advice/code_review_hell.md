# Совет: ад код-ревью (rule_id: ADV-REVIEW)

**Суть.** PR, который висит три дня, демотивирует сильнее, чем сложная задача. Ревью — это про скорость обратной связи, а не про вкусовщину.

**Шаги:**
1. Делай PR мелкими: 200 строк ревьюят за чай, 2000 — откладывают «на потом».
2. Запрашивай ревьюера явно и по имени, с мягким дедлайном («посмотри до завтра?»).
3. Дай контекст в описании: что, зачем и на что смотреть в первую очередь.
4. Разводи комментарии на blocking и nit — мелочи не должны держать мерж.

**Варианты по ситуации:**
- Тебя топят в нит-пиках по стилю — вынеси стиль в линтер/форматтер, чтобы спорил робот, а не люди.
- PR висит, потому что большой — разрежь его на серию мелких, ревью пойдёт быстрее.
- Ревьюер пропал — пингани один раз вежливо, потом попроси другого, а не сиди в ожидании.

**Когда эскалировать:** ревью системно стопорится днями и это норма команды — обсуди SLA на ревью с тимлидом, это процесс, а не твоя вина.
